## 格式基线（以参考剧本为准，覆盖 v9.2 默认）

### 场景头格式

```
第X集
编号 日/夜 内/外 场景名
人物：A，B，C
服装：A（描述），B（描述）
道具：X，Y，Z
【场景概述——方括号，不写"场景概述"四个字，直接写概述内容】
```

要点：
- 编号用三位数（001/002...）
- 人物用逗号分隔
- 服装每个角色单独括号标注
- 场景概述用【】，一句话说清场景状态

### 动作/对白/OS/特效 格式

```
△动作描写（△开头）
角色名：对白
角色名（情绪）：对白（情绪标注可用，括号内放情绪或小动作）
角色名os：内心独白
角色名（情绪OS）：带情绪的内心独白（参考剧本实际混用，两种都接受）
角色名vo：画外音（偶尔用，非主角在场时传递信息）
△【特效】特效描述
△信息：能力透视/系统面板类信息展示
```

### 集尾格式

- 不用 `---` 分隔场景（参考剧本无此用法）
- 不用 `（第X集完）` 标记
- 集与集之间：新场景用新场景头，续场直接△接续

### 续场规则

当一集在中途结束时，下一集开头如果是同一场景的继续：
- 不再写场景头
- 直接用 `△` 动作接续
- 只有当场景/时间/地点改变时才写新场景头

### 字数目标

- 每集约 650 字
- 每集约 1 分钟阅读量
- 不按场景数约束，按字数约束

### 钩子类型池

每集结尾硬钩子，类型池：
- 能力升级 / 暴富 / 打脸 / 身份反差 / 新危机 / 新发现 / 新人物登场 / 悬念 / 狠话 / 关系升级

连续 3 集不得重复同类型。

## 对白基线

- 每句 5-20 字为主，情绪爆发可破到 30+ 字
- 口语短句，直接。可用脏话：`tm的` `老子` `孙贼` `臭要饭的` `妈的` `卧槽`
- 每句对白必须推进剧情或展示人物或制造冲突
- 情绪标注前置可用：`（愣神）` `（咬牙）` `（不屑）` `（惊呼）` `（尴尬）` `（小声）`
- 对白括号内可夹动作：`（颠颠炉子）还挺沉。`

## 爽点基线

- 每集 ≥ 3 个爽点
- 类型池：捡漏 / 暴富 / 打脸 / 能力升级 / 身份反差 / 实力碾压 / 财富碾压 / 信息碾压
- 散开布置，最强爽点放结尾前
- 爽点通过 OS 传达快感：`卧槽，十五万！` `妈的，该死的资本家！`

## OS 基线

- OS是短剧标配，自然使用，不设数量硬指标
- 用途：爽感宣言 / 能力解读 / 吐槽 / 态度表态 / 战略判断
- 短而锋利，不写长段心理

## 与 v9.2 CLAUDE.md 的关系

本文件是格式层和执行层的基线。v9.2 的三条铁律（格式即法律/禁AI腔/心里装着老张）和禁词清单仍然有效。本文件的格式规则覆盖 CLAUDE.md 中的格式示例——以本文件为准。
