# Agent 2 Gotchas — 高频踩坑速查

> 每段大纲生成后必查。从 agent_outline_construction.md 移出以减少主文件体积。

| # | 陷阱 | 正确做法 |
|---|------|---------|
| 1 | 分段模式下修改前段大纲后，后段已确认内容出现连锁冲突（伏笔回收/情绪曲线/人物弧光） | 修改任一段后必须扫描所有后续段，输出"连锁影响报告"。不无声断裂 |
| 2 | 钩子类型轮换检测只看连续 3 集——A/B/A/B 交替的 4 集会绕过检测 | 额外检查：5 集窗口内同一类型出现 ≥ 3 次也标记警告 |
| 3 | 情绪弹簧"中性收尾"最隐蔽形式：结尾是安静对话场景 | 任何不以"悬念/反转/危机/情感/信息差"五钩子之一结束的集，视为疑似中性收尾 |
| 4 | 金手指"可拍摄"规则最常违反：光屏/浮空文字/瞳孔变色 OK；"他感到力量涌入"不 OK | 逐条金手指描述自问：摄影师能不能不提问直接拍？ |
| 5 | 伏笔计划过于模糊（"第 30 集揭示真相"但未指定揭示方式） | 每条伏笔必须指定：埋设方式(对话/道具/事件) + 回收方式 + 回收集数 |
| 6 | 人物 ≤ 6 人——临时角色（服务员/路人/打手）不计入，但反复出现的临时角色必须计入 | 出现 ≥ 3 集的任何有名/有对白角色 → 计入人物总数 |
| 7 | 过度依赖赛道套路——大纲读起来像"战神流模板填空"，每个单元都是"挑衅→打脸→新挑衅"循环 | 每10集单元至少1处反套路设计（观众预期A，实际给B但合逻辑） |
| 8 | 反派降智——为了让主角赢，反派在关键节点做出不符合其智商的决策 | 反派每次失败必须是被主角能力碾压，不是反派自己犯蠢。检查每个反派失败节点 |
| 9 | 角色信息不对称表维护失败——大纲阶段设计了角色间的信息差，但后续段落中角色表现出不该有的"全知" | 每段大纲完成后对照 ANCHOR.md 第六节信息不对称表：每个角色当前知道什么？不知道什么？ |
| 10 | 中期角色动机漂移——第 50-70 集角色的核心驱动力与北极星偏离 | 每段开始前重读北极星，确认每个角色的当前行为能否追溯到北极星方向。偏离 ≥ 2 度的次要角色 → 修正或标注为"支线" |
