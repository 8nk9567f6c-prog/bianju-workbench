# Agent 3 Gotchas — 高频踩坑速查

> 每集/每阶段写完后必查。从 agent_content_expansion.md 移出以减少主文件体积。

| # | 陷阱 | 正确做法 |
|---|------|---------|
| 1 | `validate_episode` 只检查单集格式——跨集问题（道具先出现后铺垫/伏笔跨集矛盾/角色知识跳跃）不会报警 | 每阶段结束必须跑 `validate_multi_episodes`，不跳过 |
| 2 | `auto_fix.py` 过度删除连接词 → 对白变电报体（全是短句无衔接） | auto_fix 后人工抽读前 3 集对白——如果句句 < 5 字且无衔接，回滚并手动逐句修 |
| 3 | `check_emotion_spring` 只检测结尾压/放状态——中段情绪中性（如 30-45s 爽点位置放了空场景描写）不触发报警 | 额外人工检查：每集第 30-45s 区段是否有情绪事件 |
| 4 | MCP 全量扫描通过但是——架空标注只存在于第 1 集，后续集未标注 | 每集独立检查：涉及修仙/系统/异能/天师的集，【】中必须有"架空"二字 |
| 5 | 道具预罗列时写了"一把刀"，正文用了"匕首"——名称不一致，MCP 按字符串匹配会漏检 | 道具名称逐集保持一致。首次出现时在预罗列中固定中文名称 |
| 6 | 散文质感中的"感官 ≥ 2 种"——连续动作场景易退化 | 每场景视觉 + 至少一种其他感官（听觉最易加：脚步声/风声/呼吸声） |
| 7 | 修改某一集后只跑了该集的 `validate_episode`，但修改影响前后衔接 | 任何修改后必须检查：前集结尾钩子 ↔ 本集开头 + 本集结尾钩子 ↔ 后集开头 |
| 8 | 对白信息密度不足——每句台词"三不沾"（不推情节/不揭人设/不制造化解冲突）仍留在剧本中 | 写完每集后"砍掉一半台词"测试：删掉一半对白，剧情仍完整 → 删。三不沾测试 + 删减测试双重过滤 |
| 9 | 爽点模式疲劳——同一爽点类型（打脸/身份揭穿/实力碾压）连续 3 集以上重复，观众产生抗体 | 建立爽点轮换日志：每集记录爽点类型+触发方式，连续 2 集同类型 → 第 3 集强制更换 |
| 10 | 情绪弹簧"假压假放"——表面压（结尾危机）但信息无增量（下一集轻松化解），或表面放（爽点）但主角未付出对等代价 | 压后检查：下集解压方式是否消耗了真实资源/关系/信息？放后检查：本集释放是否建立在主角有代价的选择之上？ |
| 11 | AI 标记词频率过高——"仿佛/忽然/竟然/不禁/宛如/猛地/难以置信"等词每集超过 3 次，剧本读起来像 AI 生成 | 每集写完后搜索这些标记词，超 3 次 → 改写。目标：每 600 字 ≤ 1 个 AI 标记词 |
| 12 | 角色信息越界——角色 A 知道了只有角色 B 在场时才发生的事，或角色对没见过的事件做出精确反应 | 每场戏三问：这个角色当时在场吗？事后谁告诉了ta？以ta的信息量能做出这个判断吗？ |
| 13 | 写作风格跨集漂移——第 1 集冷峻短句，写到第 50 集变成温情长句，风格指纹无声丢失 | v7.1 自动化：每集 `check_text_variety --baseline "作品/{项目}/剧本/剧本-第1-3集.md"` 自动对比基线，偏差 > 20% → exit 1（警告），> 30% → exit 2（阻断） |
| 14 | 动作描写的摄影机视角重复——连续 3 场戏都是"全身 → 面部特写"同一机位描述模式 | 建立机位轮换意识：全身/中景/特写/过肩/俯角/跟拍，每集至少 3 种机位变化 |
| 15 | 段落等长——每段描写都是 2-3 句，形成单调的"块状"视觉节奏 | 节奏变化检查：短句爆破(1句独立成段) ≥ 2处/集 + 画面停顿(单句成段) ≥ 1处/集 |
| 16 | 后期角色动机漂移——第 70 集后角色的选择逻辑与第 10 集设定的人设矛盾 | 每 20 集抽取角色关键选择，对照 ANCHOR.md 人物速查表验证一致性 |
| 17 | v7.1 后期读者脱钩——第 70 集后 Critic 情绪真实度持续 < 7 且 Reader-Sim 吸引力 < 6，但单集 MCP 全通过 | 每 10 集（70-100 段）强制触发 Reader-Sim + Critic，发现连续 3 集脱钩 → 回到 ANCHOR 人物速查表重验角色动机链（Gotcha #16 联动） |
| 18 | v7.1 一集塞多件事——表面只写一个场景但对话中暗藏 AB 两条线推进，实质分散 | 写完每集后一句话概括本集"唯一事件"——如果概括不出来或需要"和"字连接 → 事件过多，拆分到多集 |
| 19 | v7.1 场面调度惯性——习惯性写"他站起身走到窗前转身凝视窗外"等复杂调度 | 每处动作描写检查：是否可删？是否可替换为一句对白？| 超过 2 个连续肢体动作 → 砍掉只保留 1 个或替换为对白 |
| 20 | v7.1 伏笔忘记回收——埋设伏笔后写到下一集忘了揭示 | 埋设伏笔时立即在 hooks_ledger.json 中标注"deadline: 下一集"，写下一集前先读取 hooks_ledger 确认是否有待回收伏笔 |
