## 作品管理体系

**作品目录**：`作品/` 文件夹为所有创作项目的总根目录。每个新项目在 `作品/` 下创建独立子文件夹。
```
作品/
├── _模板/              ← 新项目模板（含子目录结构）
├── 我的妈呀是仙王/      ← 已有项目
└── (新项目)/
    ├── 大纲/
    ├── 剧本/
    ├── 人物/
    ├── 审核/
    ├── 复盘/
    └── 导出/
```

**新建项目时**：自动在 `作品/` 下创建项目文件夹，复制 `_模板/` 子目录结构。

**版本管理铁律**：`/自检` 或 `/修改` 后发现不满意版本 → 立即删除旧文件，仅保留最新通过自检的版本。禁止在项目文件夹中堆积 `_v1/_v2/_old` 等历史版本。Git 负责版本回溯，文件夹只保留当前最佳版本。

### 项目初始化
- `/新项目` — 引导收集：①简介 ②频向 ③赛道 ④平台 ⑤情绪机制 ⑥付费卡点。**在 `作品/` 下创建项目子文件夹**。创建后询问素材导入(A.上传 B.粘贴 C.口述 D.直接对标)。
- `/导入` — 支持 .txt/.docx/.md/粘贴/口述。输出：来源/类型/规模/题材匹配/改编判断。
- `/检测` — 七维分析：人物+情节+世界观+情绪价值+爽点+钩子潜力+改编可行性(1-10)。
- `/对标` — **不可跳过**。联网搜索同赛道 SS/S+ 爆款 3-5 部，**8 基因拆解**(钩子/节奏/情绪/人物/爽点/台词/付费/切片)每项标注可复制指数(1-10) + **v5.4 风格指纹提取**(开场模式/打脸频率/对白风格/情绪弹簧节奏/付费卡点间距/切片黄金秒数)。输出：赛道基因图谱+风格指纹报告+差异化切入点+S+可行性结论+风险预警。**使用 8 Agent 并行拆解。**

### 前期策划
- `/备忘` — 四大模块：金手指(激活/限制/升级/代价)/战力体系/行为准则/双反派(价值观反派=主角镜像+利益反派=具体冲突，简单清晰)。含游戏化规则。
- `/简介` — 200字北极星：设定+冲突+关系+爽点+结局。所有审核大纲一致性以此为准。
- `/人物` — ≤8人，六段式：基本信息/内在特质/背景关系/人物弧光(四阶段)/执念/切片潜力。含角色能力映射表。
- `/大纲` — Save the Cat! × 100集框架。三幕(1-30建置/31-75对抗/76-100结局)+情绪曲线+欠条设计(存入/催收集数)+付费节点+双轨节奏+10单元规划。**输出后必须停止等确认。**
- `/梗概` — 100集每集2-3句，标注：钩子类型/爽点/付费情绪强度/不可逆变化/情绪锚点(爽虐暖悬燃)/单元功能。

## 并行 Agent 审核系统

审核命令（`/审核1` `/审核2` `/审核3` `/审核50` `/终审`）自动启动 5 个并行 Agent：

| Agent | 职责 |
|-------|------|
| Agent 1: 剧情大纲 | 是否偏离大纲和"北极星"？偏离是"合理优化"还是"需修正"？10集单元闭环？**对照 ANCHOR.md 单元路线图+伏笔总账。** |
| Agent 2: 人物设定 | 角色行为符合人设？有无"为剧情需要"违反人设？信息一致性？弧光进度？双反派均衡？**对照 ANCHOR.md 人物速查表。** |
| Agent 3: 已有逻辑 | 金手指规则一致性/伏笔清单(已埋/已回收/待回收)/道具与预罗列一致性/时间线/代价兑现/战力体系。**对照 ANCHOR.md 伏笔总账+付费卡点弹簧。** |
| Agent 4: 平台红线 | 逐条对照9条禁止+敏感内容规避+架空标注+正义结局+版权侵权 |
| Agent 5: 格式规范 | 场景编号连续性/人物服装道具罗列/【】完整(站位+动作+环境+氛围)/△正确/对白格式/≥500字/≥3节拍/道具一致性 |

**各阶段追加检查**：
- 审核2(1-10集)：10集闭环？一卡情绪积累？三类欠条存款+复利？钩子轮换？
- 审核3(1-30集)：三个闭环？人物弧光(生存依附→力量觉醒→动态对抗)？二卡情绪催收？伏笔回收率？
- 审核50(1-50集)：中点反转？5个单元闭环？弧光到"动态对抗"？伏笔回收≥60%？情绪银行健康度？
- 终审(1-100集)：北极星一致性？10单元全部闭环？弧光收束？伏笔回收≥95%？金手指全额兑现？正义结局？终场vs开场？总字数≥5w？

**汇总报告**：五维总览表 + 详细问题清单 + S+评分影响 + 修改优先级(高/中/低) + 总体结论(通过/有条件通过/不通过)

**审核后强制**：更新 ANCHOR.md 的"七、当前写作锚点"（当前阶段/已完成集数/上次结尾/待解决冲突/下一卡点/本段核心任务）+ "五、伏笔总账"（新埋/已回收/状态更新）。

**对标分析 8 Agent 并行**：钩子/节奏/情绪/人物/爽点/台词/付费/切片基因同步拆解。

## 算法工程学（核心参数速查）

**平台铁律**：前3秒=60%流量 / 完播率≥40%=SS级 / 连续10s无新信息=死亡 / 点赞≥5% / 分享≥3%

**钩子密度**：1-3集20s/4-10集30s/11-30集45s/31-100集45-60s。五类型(连续3集不得重复)：悬念/反转/危机/情感/信息差。前10集端钩子全A级，付费节点前A+级，单元高潮A+级。

**节奏五节点**：0-1.5s视觉锤(0.2s出字幕)→2-5s立场锚(3s认知闭环)→10-20s爆破点→30-45s爽点/反转→集末3-5s终极钩子。切分：每3-5s视觉变化/每5-8s模式打断/连续10s无新=死亡/连续两段纯对话=死亡。

**情绪银行**：三类欠条(人物缺口/关系缺口/真相缺口)×三层机制(存款前3集→复利4-50集→催收付费节点)。付费四卡点：10集(一卡)/30集(二卡)/50集(三卡)/75集(四卡)。付费前3集：N-2存款→N-1复利→N暴雷。每集标注情绪锚点(爽虐暖悬燃)+波动≥3次。

**红果必爆七招**：招1对抗式开场(0-3s)→招2：30s爆破点→招3：60s第1爽点→招4：90s第2反转→招5：120s第2爽点→招6：150s终极钩子→招7：180s倒计时。

**游戏化系统**：面板须可被摄影机拍到(光屏/浮空文字/瞳孔变色)。数值是剧情引擎。一致性铁律：规则一旦建立全剧不得矛盾。
