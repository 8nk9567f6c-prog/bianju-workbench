# 编剧工作台 v4.1 — 网页端使用说明

## 访问方式

### ⚠️ 重要：不能直接双击打开 index.html
由于浏览器安全策略，`file://` 协议下 `fetch('data.json')` 会失败，导致网页无法加载数据。

### 电脑端（推荐方式）
双击运行 `启动服务器.bat`，浏览器打开 `http://localhost:8080`

### 手机端（同一WiFi下访问）
1. 双击运行 `启动服务器.bat`
2. 屏幕会显示手机访问地址，例如 `http://192.168.1.100:8080`
3. 手机连接同一 WiFi，浏览器输入该地址
4. 在手机浏览器菜单中选择「添加到主屏幕」即可作为 PWA 使用

### 远程访问（随时随地）
免费部署平台（任选其一）：
- GitHub Pages：把 web 文件夹上传到 GitHub 仓库，开启 Pages
- Vercel：导入 GitHub 仓库，自动部署，获得 `xxx.vercel.app` 域名
- 腾讯云静态网站托管：上传 web 文件夹即可

## 手机打不开的常见原因

| 问题 | 原因 | 解决 |
|------|------|------|
| 页面空白 | 直接双击打开了 index.html（file:// 协议） | 必须通过 `启动服务器.bat` 启动本地服务器 |
| 数据加载失败 | fetch 被浏览器 CORS 策略阻止 | 同上，使用 HTTP 服务器 |
| 手机找不到页面 | 电脑和手机不在同一 WiFi | 确保同一网络，检查电脑防火墙是否开放 8080 端口 |
| PWA 安装不了 | Service Worker 需要 HTTPS 或 localhost | 电脑用 localhost 可安装，手机需部署到 HTTPS 服务器 |
| 显示 v4.0 而非 v4.1 | 浏览器缓存了旧版本 | 清除浏览器缓存或使用无痕模式打开 |

## 数据联动机制

网页端通过读取 `data.json` 获取所有规则和工作流数据。

### 更新网页端数据

当你修改了编剧工作台的底层规则（CLAUDE.md），按以下步骤同步网页：

1. 编辑 `C:\Users\17928\Desktop\编剧工作台\web\data.json`
2. 找到对应的规则分类
3. 修改或新增规则条目
4. 更新 `"updated"` 日期
5. 保存文件

网页端有自动缓存机制：
- 打开网页时自动加载最新 data.json
- 点击工作台页面的「刷新数据」按钮可手动刷新
- 离线时自动使用浏览器 localStorage 缓存的数据

### 文件结构

```
web/
├── index.html      ← 网页应用主文件
├── data.json       ← 规则数据库（修改此文件即可更新网页内容）
├── sw.js           ← Service Worker（离线缓存）
└── manifest.json   ← PWA 配置
```

### 数据联动关系

```
CLAUDE.md  ←── 编剧工作台核心配置（Claude Code 读取）
    │
    └── 手动同步 ──→  web/data.json  ←── 网页应用规则数据
                           │
                           └── 自动加载 ──→  index.html 网页端显示
```
