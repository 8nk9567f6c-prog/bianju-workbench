## 编排水线 —— `/流水线` 命令

一键串联三智能体。用户只需提供创作意图，系统自动依次调用三个智能体，每阶段停止等确认：

```
/流水线
  阶段 1: 调用 Agent 1（选题调研）→ 输出报告 → 停止等确认
  阶段 2: 调用 Agent 2（大纲搭建）→ 输出 ANCHOR.md → 停止等确认
  阶段 3: 调用 Agent 3（内容扩充）→ 五阶段写作 → 每阶段停止等确认
  最终: /导出 汇总 Word
```

## 单独调用

每个智能体也可以独立调用（例如只需要调研、只需要大纲）：

| 命令 | 智能体 | 使用场景 |
|------|--------|----------|
| `/选题调研` | Agent 1 | 已有创意想法，需要验证市场可行性 |
| `/大纲搭建` | Agent 2 | 已有确定选题，需要构建完整大纲 |
| `/内容扩充` | Agent 3 | 已有完整大纲，需要产出剧本正文 |
| `/流水线` | 全部 | 从零开始，一键跑通全流程 |

## 智能体文件结构

```
agents/
├── README.md                       ← 本文件（系统总览 + 编排指南）
├── FOUNDATION.md                   ← ★ 底层创作宪法——所有智能体强制预加载
├── agent_topic_research.md         ← Agent 1 完整提示词
├── agent_outline_construction.md   ← Agent 2 完整提示词
└── agent_content_expansion.md      ← Agent 3 完整提示词
```

每个 Agent 提示词文件均包含：
- **Role Definition** — 角色定位和使命
- **Domain Knowledge** — 专精领域知识（不重复 DNA 已有内容）
- **Input Contract** — 输入要求
- **Workflow** — 分步工作流（含门禁检查点）
- **Output Contract** — 输出要求和格式
- **Quality Gate** — S+ 评分门禁
- **Redlines** — 绝对不能做的事

## 设计原则

1. **不替换现有命令** — 智能体编排现有 35 斜杠命令，增加质量门禁和交接协议
2. **每阶段等确认** — 创作类工作 S+ 级需人工判断，智能体完成每阶段后停止等用户确认
3. **Markdown 交接** — 智能体之间通过项目目录下的 .md 文件传递数据（调研报告 → ANCHOR.md → 剧本正文）
4. **DNA 独立加载** — 每个智能体独立读取 CORE_CREATIVE_DNA.md，确保跨会话一致性
5. **门禁不妥协** — 任何阶段 S+ < 85 = 停止推进，迭代至达标或手动跳过
